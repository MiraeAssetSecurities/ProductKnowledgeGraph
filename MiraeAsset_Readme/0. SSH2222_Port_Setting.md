# Ubuntu에서 SSH를 22번 + 2222번 포트로 동시에 리슨하기

## 📌 문제 상황

- `/etc/ssh/sshd_config`에 `Port 2222`를 추가함
- `sudo sshd -T`로 보면 `port 22`, `port 2222` 모두 정상 인식
- 그러나 `sudo ss -tlnp | grep ssh` 실행 시 22번만 리슨
- 로그에도 2222 관련 에러가 없음

---

## 🔎 원인

Ubuntu 최신 버전에서는 **systemd socket activation (`ssh.socket`)**이 활성화되어 있을 수 있음.

이 경우:

- 실제로 포트를 리슨하는 주체는 `sshd`가 아니라 `systemd`
- `ssh.socket`에 정의된 `ListenStream` 포트만 열림
- `sshd_config`의 `Port` 설정은 무시됨
- 따라서 2222는 실제로 열리지 않음

---

# ✅ 해결 방법 1 (권장): ssh.socket에 2222 추가

## 1️⃣ 현재 상태 확인

```bash
sudo systemctl status ssh.socket
sudo systemctl is-enabled ssh.socket
```

active (listening)이면 socket activation이 사용 중임.

## 2️⃣ ssh.socket override 설정
sudo systemctl edit ssh.socket

아래 내용을 입력:
```bash
[Socket]
ListenStream=
ListenStream=22
ListenStream=2222
```

### ⚠️ 중요
ListenStream= 빈 줄은 기존 설정을 초기화(reset)하기 위한 것임.
이 줄이 없으면 설정이 제대로 덮어쓰이지 않을 수 있음.

## 3️⃣ 설정 적용
```bash
sudo systemctl daemon-reload
sudo systemctl restart ssh.socket
sudo systemctl restart ssh
```

## 4️⃣ 리슨 확인
```bash
sudo ss -tlnp | grep ssh
```

정상 출력 예시:
```bash
LISTEN 0 4096 0.0.0.0:22
LISTEN 0 4096 0.0.0.0:2222
```

### ✅ 해결 방법 2: socket activation 비활성화

sshd가 직접 포트를 리슨하도록 변경하는 방법.
```bash
sudo systemctl disable --now ssh.socket
sudo systemctl restart ssh
```

이후 /etc/ssh/sshd_config의:
```bash
Port 22
Port 2222
```

설정이 그대로 적용됨.

## 🔐 EC2 사용 시 추가 설정

서버 내부에서 2222가 리슨 중이어도 외부 접속은 별도 설정이 필요함.
### 1️⃣ Security Group Inbound 규칙 추가
```bash
Type: Custom TCP
Port: 2222
Source: 허용할 IP 범위
```

### 2️⃣ UFW 사용 시
```bash
sudo ufw allow 2222/tcp
sudo ufw reload
```

### 📎 디버깅 명령어 모음
sshd가 인식하는 포트 확인
```bash
sudo sshd -T | grep -i '^port'
```

socket 설정 확인
```bash
sudo systemctl cat ssh.socket
```

2222 포트 충돌 확인
```bash
sudo lsof -iTCP:2222 -sTCP:LISTEN -nP
```
